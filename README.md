# Сайт-подарунок на річницю ❤️

Готовий каркас сайту. Залишилось додати твої файли — і все запрацює.

## Що зробити перед запуском

1. **Фото.** Поклади всі фотографії у папку `assets/photos/` з точно такими іменами,
   як у списку нижче (вони вже прописані в `js/data.js`):
   - `mainhero.jpg` — головне фото Hero-секції
   - `funnya1.jpg` … `funnya7.jpg` — твої смішні фото (funnya7 — пасхалка)
   - `funnyp1.jpg` … `funnyp24.jpg` — "Галерея компромату"
   - `together1.jpg` … `together16.jpg` — спільні фото
   - `seriousa1.jpg`, `seriousa2.jpg`, `seriousa3.jpg` — серйозний блок
   - `goodphoto.jpg` — "З тобою я виріс"
   - `krevetkaa.jpg` — "Креветка року"
   - `eae1.jpg` … `eae15.jpg` — маленькі фото-пасхалки (сердечка по сайту)

2. **Музика.** Поклади пісню у `assets/music/song.mp3`.

3. **Дата стосунків.** Відкрий `js/data.js` і встанови правильну дату в
   `RELATIONSHIP_START_DATE` (рядок на початку файлу).

4. **Підпис у листі (за бажанням).** У `index.html` знайди рядок із
   підписом у фінальному блоці "Лист для тебе" і зміни текст, якщо потрібно.

5. **Тексти пасхалок (за бажанням).** Підписи до кожного маленького сердечка
   редагуються в `js/easterEggs.js`, у масиві `easterEggs`.

## Пасхалки (маленькі сердечка)

По всій сторінці розкидано 15 напівпрозорих сердечок ❤️, що плавно
пульсують. Клік відкриває маленьке фото-спогад і збільшує лічильник
`❤️ 0/15` у верхньому лівому куті. Коли знайдено всі 15 — з'являється
святкова модалка та конфеті. Прогрес зберігається в `localStorage`
браузера, тож при повторному відкритті сайту вже знайдені сердечка не
показуються знову.

## Як запустити

Просто відкрий `index.html` у браузері — все запрацює локально, без сервера
і без збірки.

Якщо браузер блокуватиме завантаження локальних файлів (буває з деякими
версіями Chrome через CORS-обмеження для `file://`), можна підняти простий
локальний сервер з папки проєкту:

```bash
python3 -m http.server 8000
```

і відкрити `http://localhost:8000` у браузері.

## Структура проєкту

```
anniversary/
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── data.js        — увесь контент: тексти, фото, дата, налаштування
│   ├── gallery.js      — генерація розділів та фото-сітки з data.js
│   ├── modal.js         — модальне вікно перегляду фото
│   ├── music.js         — кнопка "Наша пісня"
│   ├── easterEggs.js    — 15 прихованих сердечок по сторінці
│   └── main.js           — лічильник часу, AOS, плавний скрол, лист
└── assets/
    ├── photos/        — сюди фото
    └── music/          — сюди song.mp3
```

## Як додати новий розділ

Усе живиться з `js/data.js`. Щоб додати ще один розділ історії — додай новий
об'єкт у масив `MEMORIES` (формат `type: "gallery"`, `"gallery-easter-egg"`
або `"single"` — приклади вже є у файлі). Верстка згенерується сама.
